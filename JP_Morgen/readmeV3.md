# V2 → V3 升级方案

---

## V2 vs 主流量化 Transformer 的差距

| 维度 | 你的 V2 | 主流做法 (Quantformer / MASTER / GrifFinNet) |
|------|--------|------------------------------------------|
| **Attention 维度** | 只有时间维度（每只股票独立看自己的历史） | 时间 + 横截面双维度（股票之间也互相看） |
| **参数量** | ~77K | 1M - 10M |
| **d_model** | 96 | 128 - 256 |
| **因子数** | 15-24 个技术因子 | 50-100+ 因子（含基本面 + 另类数据） |
| **Positional Encoding** | 可学习参数（随机初始化） | Time2Vec（连续时间编码） |
| **损失函数** | Ranking + MSE | Ranking + MSE + IC 正则化 |
| **股票间关系** | 无 | 行业图、持股关系图、相关性图 |
| **训练数据** | ~20K 样本 | 500K - 5M+ 样本 |

## V3 的 5 个核心升级

### 升级 1：双 Attention — Temporal + Cross-Sectional

V2 的 Transformer 只在时间维度做 self-attention：每只股票独立看自己过去 20 天的数据。但主流模型如 MASTER 会交替使用两种 attention：

```
Temporal Attention:  同一只股票的不同天互相看
  → "NVDA 今天的模式像不像 10 天前？"

Cross-Sectional Attention: 同一天的不同股票互相看
  → "今天 NVDA 和 AMD 的模式有什么关系？"
```

这是最关键的升级。你的 V2 完全缺少横截面信息——模型不知道"今天整个半导体板块都在涨"这种跨股票信号。

### 升级 2：Time2Vec 位置编码

V2 用随机初始化的可学习位置编码：

```python
# V2: 随机初始化，模型需要从头学习时间模式
self.pos_enc = nn.Parameter(torch.randn(1, 100, d_model) * 0.01)
```

Time2Vec（2019 论文）用 sin/cos 函数编码时间，既能捕捉周期性模式（周内效应、月末效应），又保留了可学习性：

```python
# V3: Time2Vec — 一部分固定周期 + 一部分可学习
# t2v(t) = [ωt + φ, sin(ω₁t + φ₁), sin(ω₂t + φ₂), ...]
```

### 升级 3：模型规模提升

```
V2: d_model=96,  nhead=4,  layers=3  → ~77K params
V3: d_model=192, nhead=6,  layers=4  → ~800K params（10x）
```

head_dim 保持 32（192/6=32），比 GPT 的 64 小一些，适合金融数据的较低维度。

### 升级 4：IC 正则化损失

在 Ranking Loss + MSE 的基础上，加入 IC（Spearman 相关）作为额外的优化目标：

```
Total Loss = 0.5 × Rank + 0.2 × MSE + 0.3 × (1 - IC)
```

直接优化 IC 让模型更专注于排名的秩相关性。

### 升级 5：基本面因子（从 yfinance 获取）

yfinance 可以免费获取部分基本面数据（PE、PB、市值、ROE 等），不需要付费数据源。

## 一、策略的三层架构

一个完整的量化策略 = Alpha 信号 + 投资组合构建 + 执行规则。模型只负责第一层。

### 第一层：Alpha 信号（模型做的事）

```
输入: 每只股票过去 20 天的 30 个因子
       │
       ▼
  Dual-Attention Transformer
  ├── Temporal Attention: 学习时间维度的模式
  │     "NVDA 今天的因子组合像不像 10 天前大涨之前的模式？"
  ├── Cross-Sectional Gate: 学习股票之间的关系
  │     "整个半导体板块都在放量，NVDA 是不是会跟着涨？"
  └── Temporal Attention Pooling: 学习哪天更重要
        "最近 3 天的信号比 15 天前的更重要"
       │
       ▼
输出: 每只股票未来 5 天的预期收益排名
```

模型的核心能力是**排名**——不需要精确预测 NVDA 会涨 2.3%，只需要判断"NVDA 会比 INTC 涨得更多"。这就是为什么我们用 Ranking Loss 而不是纯 MSE。

### 第二层：投资组合构建（人为设定的规则）

| 规则 | 当前设定 | 含义 |
|------|---------|------|
| 多空结构 | Long Top 3 + Short Bottom 3 | 市场中性（不赌大盘方向） |
| 权重分配 | 等权（每只股票相同金额） | 最简单的方式 |
| 行业约束 | 无 | 可能导致行业集中风险 |
| 最大持仓 | 无限制 | 理论上一只股票可以占 33% |

这些规则不是模型学到的，而是策略设计者选择的。不同选择会产生不同效果：

```
变体 A: 只做多 Top 5（纯多头）
  → 牛市表现好，熊市会亏
  
变体 B: Long Top 3, Short Bottom 3（当前设定）
  → 市场中性，牛市熊市都能赚（如果排名准确）

变体 C: 按预测值大小加权分配
  → 对预测最有信心的股票投更多钱
  
变体 D: 加行业中性约束
  → 科技做多 2 只 + 做空 1 只，能源做多 1 只 + 做空 2 只
  → 避免纯做多科技/纯做空能源的行业偏差
```

### 第三层：执行规则

| 规则 | 当前设定 | 影响 |
|------|---------|------|
| 调仓频率 | 每 5 个交易日 | 频率越高 → 信号越新鲜，但成本越高 |
| 交易成本 | 0.1% 双边 | 只对换手部分收费 |
| 滑点模型 | 无 | 真实中大单会推高买入价 |
| 资金规模 | 无限制（隐含假设资金很小） | 大资金需要考虑市场冲击 |

## 三、资金容量（Strategy Capacity）

### 为什么策略有资金上限？

核心原因：你的交易本身会影响市场价格。

```
你想买入 SNAP（日均成交额 ~500 万美元）

资金 10 万 → 买入 1.7 万 → 占日均成交 0.3% → 价格几乎不变
资金 100 万 → 买入 17 万 → 占日均成交 3.4% → 价格被推高约 0.2%
资金 1000 万 → 买入 170 万 → 占日均成交 34% → 价格被推高 > 1%
                                                     ↑ 还没买完价格就涨上去了
```

这叫 **Market Impact（市场冲击）**，是资金容量的核心限制。

### 市场冲击的简化模型

学术界常用的 Square-Root 模型：

```
Impact ≈ σ × √(Q / ADV)

σ   = 日波动率
Q   = 你的交易量（美元）
ADV = 该股票的日均成交量（美元）

例子：
  AAPL: σ=1.5%, ADV=100亿, Q=50万
  Impact = 1.5% × √(50万/100亿) = 1.5% × 0.007 = 0.01% → 几乎没有

  SNAP: σ=3%, ADV=500万, Q=50万
  Impact = 3% × √(50万/500万) = 3% × 0.316 = 0.95% → 很大！
```

### 不同资金规模的影响

| 资金规模 | 每只股票分配 | 市场冲击 | 可交易股票 | 预期夏普衰减 |
|---------|-----------|--------|---------|-----------|
| 10 万美元 | ~1.7 万 | ≈ 0 | 所有 65 只 | 无 |
| 100 万美元 | ~17 万 | 0.05-0.2% | 所有 65 只 | 5-10% |
| 1000 万美元 | ~170 万 | 0.2-1% | 需排除小盘股 | 20-40% |
| 1 亿美元 | ~1700 万 | 1-5% | 只能大盘股 | 50-70% |

### 参与率的安全线

```
参与率 = 你的交易量 / 日均成交量

< 1%   → 安全，几乎无冲击
1-5%   → 可接受，但有明显成本
5-10%  → 危险，需要分多天执行
> 10%  → 不可行，会严重影响价格
```

JP 摩根这样的大机构管理数百亿美元，所以它们必须：
- 交易几千只大盘股（分散资金）
- 使用算法交易（TWAP/VWAP）分散执行
- 限制每只股票的持仓上限

### 你的策略的容量估算

```
股票池: ~65 只，大部分是大盘科技/能源
持仓: 6 只（Long 3 + Short 3）
每只股票分配: 总资金 / 6

最小 ADV（股票池中最小的，如 ON Semiconductor）: ~2 亿美元/天
参与率 1% 上限 → 单只最大交易: 200 万美元
× 6 只 = 1200 万美元

保守估计策略容量: ~1000 万美元
如果只交易大盘股 (AAPL/NVDA/XOM 等): ~1 亿美元
```

## 四、将资金容量加入模型的方法

### 方法 1：在回测中加入市场冲击成本（推荐）

不改模型，只在回测的交易成本中加入冲击项：

```python
# 市场冲击 = σ × √(交易金额 / 日均成交额)
impact = stock_volatility * np.sqrt(trade_amount / avg_daily_volume)
total_cost = spread_cost + impact
```

### 方法 2：在投资组合构建中加入流动性过滤

在选股之前，先排除流动性不足的股票：

```python
# 排除日均成交额 < 1000 万的股票
liquid_stocks = [t for t in stocks if avg_volume_usd[t] > 10_000_000]
# 只在 liquid_stocks 中做排名和选股
```

### 方法 3：按流动性加权分配

给流动性好的股票更多仓位：

```python
# 按日均成交额加权
weight[i] = adv[i] / sum(adv)  # 成交额大的分配更多资金
# 而非等权: weight[i] = 1/N
```

### 课程项目建议

```
V1: "简单模型看起来赚钱"（假象）
  ↓ 诊断：前视偏差 + 无交易成本
V2: "严格评估后发现亏钱"（现实）
  ↓ 诊断：IC 为负 + 分位数反转

```

每一步都有明确的问题 → 诊断 → 解决方案 → 验证。这比直接展示一个"好结果"有说服力得多。
